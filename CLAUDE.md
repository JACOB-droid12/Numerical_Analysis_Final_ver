## Design Context

### Product
Standalone browser-based numerical analysis teaching calculator focused on machine arithmetic, error analysis, and polynomial evaluation. The app is meant to open locally with no backend or runtime setup.

### Users
Primary users are both students and instructors.

Students use the app during:
- lab exercises
- homework
- exam preparation
- self-study

Instructors may use it for:
- demonstrations
- guided examples
- quick verification of textbook-style numerical analysis problems

Users often need trustworthy results quickly and may be working under time pressure, so the interface should reduce friction and keep the main answer easy to find.

### Brand Personality
Clean, easy to use, modern.

The interface should feel:
- calm
- academically credible
- precise
- approachable under pressure

### Aesthetic Direction
Support both light mode and dark mode in future work.

The visual direction should stay focused on clarity first:
- structured, scan-friendly layouts
- strong answer hierarchy
- restrained, modern styling
- contemporary controls with durable typography

Avoid:
- flashy dashboard aesthetics
- decorative gradients or glass effects used as style-first filler
- dense card nesting
- playful or novelty-heavy visuals that compete with the math
- generic AI-generated UI tropes

### Design Principles
1. Put the mathematical answer first.
   The main computed value should be the first thing a user can find, with supporting explanation and deep detail following after.

2. Make every interaction reduce uncertainty.
   Labels, empty states, and errors should tell users what went wrong, what to enter, and what happens next.

3. Teach without slowing experts down.
   The app should support learning and interpretation, but advanced users should still be able to compute quickly.

4. Keep the visual system disciplined.
   Use consistent tokens for spacing, typography, color, and surfaces. Avoid one-off styles and repeated visual noise.

5. Design for high-stakes clarity.
   Since the app is used for labs, homework, exams, and self-study, readability, accuracy signaling, and calm presentation matter more than visual novelty.

6. Build toward accessible dual-mode use.
   Future UI work should preserve contrast, keyboard clarity, and reduced-motion support across both light and dark themes.
